Backend API Progress

2. Added `GET /config`
   - Returns current backend settings such as sample rate, chunk duration, overlap, and supported classes.

Audio Processing Progress
1. Added PCM WAV decoding support.
2. Added mono conversion from uploaded audio frames.
3. Added waveform normalization to a shared target peak.
4. Added linear resampling to the configured shared sample rate.
5. Added log-mel spectral feature generation for model-ready analysis inputs.
6. Added extracted metadata in the response:
   - sample rate
   - processed sample rate
   - number of channels
   - sample width
   - duration
   - frame count
   - processed sample count
   - normalization gain and peak values
7. Added computed audio features in the response:
   - RMS
   - peak amplitude
   - zero crossing rate
   - dominant activity ratio
8. Added compact log-mel summary values in the response:
   - spectrogram frame count
   - mel bin count
   - min, max, and mean dB
   - dynamic range
   - low, mid, and high band means
9. Added a baseline classifier module that scores the supported sound classes.
10. Replaced the old inline heuristic detections with classifier-backed predictions.
11. Optimized log-mel extraction with a `numpy`-accelerated path and kept the pure-Python path as a fallback.

Current State of the Prototype
1. The backend can accept a PCM WAV file through `/analyze`.
2. The backend now normalizes and resamples uploaded audio before feature extraction.
3. The backend now computes a log-mel spectrogram summary on the processed waveform.
4. The backend now runs a baseline classifier over extracted waveform and spectral summaries.
5. The backend returns real metadata and feature values from the uploaded file.
6. The response includes preprocessing metadata so later model inputs are traceable.
7. The detections are classifier-backed, but still rules-based rather than model-trained AI predictions.
8. The frontend has not been implemented yet beyond placeholder structure.
