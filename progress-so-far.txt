Backend API Progress
1. Added `GET /health`
   - Confirms the backend service is running.
2. Added `GET /config`
   - Returns current backend settings such as sample rate, chunk duration, overlap, and supported classes.
3. Added `POST /analyze`
   - First started as a placeholder route.
   - Then upgraded into a real WAV analysis pipeline.
4. Added `POST /process`
   - Accepts a WAV file plus suppression settings.
   - Returns a processed WAV preview using prototype class-based attenuation.
5. Added `GET /sessions`
   - Returns recent saved analysis sessions.
6. Added `GET /sessions/{session_id}`
   - Returns a saved session with original audio, analysis response, and any saved processed preview.
7. Added `POST /recordings`
   - Accepts a WAV file plus label metadata.
   - Saves validated recordings into `training/real_recordings/<split>/<label>/` or `training/real_recordings/<label>/`.
