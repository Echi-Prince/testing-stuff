MVP Task Breakdown

Completed Core Backend
1. Real WAV decoding, preprocessing, feature extraction, windowed detection merging, and session persistence are implemented.
2. The backend now exposes `GET /health`, `GET /config`, `POST /analyze`, `POST /process`, `GET /sessions`, `GET /sessions/{session_id}`, and `POST /recordings`.
3. Prototype suppression and training-recording ingest are both available through the API.

Completed Core Frontend
1. The dashboard supports file upload, browser microphone recording, analysis state, timeline review, waveform/spectrogram previews, and suppression controls.
2. Saved sessions can be reloaded from the backend.
3. Recorded microphone clips can be labeled and saved into the training dataset layout.

Current Remaining Work
1. Collect more real labeled recordings per supported class.
2. Retrain and calibrate thresholds so the trained model replaces the heuristic fallback more often.
3. Improve suppression quality beyond simple span attenuation.
4. Add broader integration coverage for frontend and end-to-end flows.
